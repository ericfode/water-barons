# Waterbarons UI Review Personas

These personas are review lenses for Waterbarons. They are not fictional target users
to appease. Each one catches a different class of failure in the browser slice.

## 1. Mira Vale, First-Run Systems Reviewer

Lens: can a new player form a correct next action in 15 seconds without reading design
docs?

Checks:

- The screen exposes one obvious next move.
- The next move explains cost, immediate effect, and delayed downside.
- Build, scout, sell, route, and policy actions use the same commit grammar.
- A player can tell whether they are inspecting, scouting, or building.
- Deselect, cancel, and recover states are visible.

Failure signals:

- The player asks "what do I do now?"
- The player clicks the map and then scrolls the side panel looking for the real action.
- The player cannot tell whether a staged action is already committed.

## 2. Cass Weller, Basebuilder Feel Reviewer

Lens: does the game feel like growing infrastructure, not moving board pieces?

Checks:

- Facilities read as physical assets with different silhouettes.
- The network reads as a connected corporate system: pipes, flow, ownership, storage.
- Valid build sites feel like expansion frontiers.
- Production pulses and storage changes are visible on the map.
- Empty space feels like unexplored basin, not blank UI.

Failure signals:

- The map reads as icons on a grid.
- The player's base does not feel more substantial after building.
- Resource changes happen only in panels.

## 3. Leona Brandt, Corporate Satire Art Director

Lens: does the interface glisten with short-sighted institutional optimism?

Checks:

- The visual language is bright, polished, and slightly too confident.
- Brass, teal, glass, ivory, and clean infrastructure cues dominate.
- Collapse pressure is presented as managed instrumentation, not horror grime.
- Social/political systems look like corporate dashboards with moral leakage.
- The world can be generated from a small tile/facility vocabulary.

Failure signals:

- The UI becomes dark, noisy, cyberpunk, fantasy, or apocalyptic grunge.
- The HUD looks like a generic admin dashboard.
- Satire appears only in text, not in materials, controls, and feedback.

## 4. Anton Reyes, Legibility And Accessibility Reviewer

Lens: can map state be read quickly under imperfect attention?

Checks:

- Terrain categories are distinguishable by shape as well as color.
- Facilities remain readable at normal zoom.
- Critical gauges stay above the fold.
- Text never depends on tiny map badges alone.
- Hover/selection states reveal exact data without cluttering every tile.

Failure signals:

- The player has to memorize color meanings.
- Important map state is lost behind transparency.
- Buttons have unclear labels or icon-only ambiguity.

## 5. Sera Holt, Multiplayer Roguelike Loop Reviewer

Lens: do cadence, pressure, and rivalry feel like the run engine?

Checks:

- Beat clocks create useful anticipation.
- Route offers feel like timed run rewards or traps.
- Policy docket choices create visible pressure before they resolve.
- Rival actions are visible enough to generate suspicion and blame.
- The run can produce "I should have done that sooner" moments.

Failure signals:

- Route selection feels like a static shop.
- Politics lives only in a side panel.
- Multiplayer state is reduced to owner colors with no rivalry pressure.
