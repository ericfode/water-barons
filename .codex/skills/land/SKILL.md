# Land

Use this only when the Linear issue is in `Merging` or the user explicitly asks to
land a PR.

## Preconditions

- `gh` CLI is authenticated.
- You are on the PR branch with a clean working tree.
- The current branch has an open PR.

## Steps

1. Locate the PR for the current branch.
2. Confirm the local gauntlet is green:

   ```bash
   npm run test
   npm run build
   ```

3. If the working tree has uncommitted changes, commit with the `commit` skill
   and push with the `push` skill before proceeding.
4. Check mergeability and conflicts against the PR base.
5. If conflicts exist, use the `pull` skill to merge the base branch, resolve
   conflicts, rerun validation, and publish the branch.
6. Gather human and bot review feedback. Address, reply to, or explicitly push
   back on every actionable comment before merging.
7. Watch checks until complete:

   ```bash
   gh pr checks --watch
   ```

8. If checks fail, inspect logs, fix the issue, commit, push, and repeat.
9. When local validation, CI, and review state are clean, squash-merge with the
   PR title/body.

## Async Watch Helper

When available, use the copied watcher to monitor review comments, CI, and head
updates:

```bash
python3 .codex/skills/land/land_watch.py
```

Exit codes:

- `2`: review comments detected;
- `3`: CI checks failed;
- `4`: PR head updated.

## Commands

```bash
branch="$(git branch --show-current)"
pr_number="$(gh pr view --json number -q .number)"
pr_title="$(gh pr view --json title -q .title)"
pr_body="$(gh pr view --json body -q .body)"

npm run test
npm run build

gh pr checks --watch
gh pr merge --squash --subject "$pr_title" --body "$pr_body"
```

## Review Handling

- Human review comments are blocking until addressed or explicitly answered.
- For inline comments, reply inline through the review-comment endpoint when
  possible.
- All GitHub comments generated by this agent should be prefixed with `[codex]`.
- Do not enable auto-merge.
